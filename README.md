## My 90‑Day Cloud Support Roadmap

### Days 1–10: Foundations
- Day 1: Linux basics, EC2 SSH connection, GitHub setup.  
- Day 2: Networking basics (IP, DNS, Security Groups).  
- Day 3: S3 bucket creation, upload, permissions.  
- Day 4: IAM users, groups, policies.  
- Day 5: EBS volumes attach/detach, snapshots.  
- Day 6: CloudWatch metrics, alarms, SNS notifications.  
- Day 7: S3 bucket policies, static website hosting.  
- Day 8: IAM roles for EC2 → S3 access.  
- Day 9: EC2 key pairs, security best practices.  
- Day 10: Linux permissions, users, groups.

### Days 11–20: Intermediate AWS
- Day 11: VPC basics (subnets, route tables).  
- Day 12: Internet Gateway + NAT Gateway setup.  
- Day 13: Launch EC2 in private subnet.  
- Day 14: CloudWatch dashboards.  
- Day 15: CloudTrail basics (audit logs).  
- Day 16: S3 lifecycle rules + versioning.  
- Day 17: IAM MFA setup.  
- Day 18: EC2 Auto Scaling basics.  
- Day 19: Elastic Load Balancer setup.  
- Day 20: Review + GitHub documentation update.

### Days 21–40: Projects
- Day 21: Build simple website on EC2.  
- Day 22: Connect EC2 → S3 for file storage.  
- Day 23: Monitor website with CloudWatch alarms.  
- Day 24: Create IAM role for app access.  
- Day 25: Document project in GitHub.  
- Day 26: Build VPC with public/private subnets.  
- Day 27: Deploy EC2 in VPC.  
- Day 28: Test connectivity with Security Groups.  
- Day 29: Add CloudWatch monitoring to VPC project.  
- Day 30: Review + GitHub update.  
- Day 31–40: Repeat with 2 more mini‑projects (mix EC2, S3, IAM, CloudWatch).

### Days 41–60: Advanced Practice
- Day 41: Troubleshooting EC2 connectivity.  
- Day 42: IAM policy debugging.  
- Day 43: S3 access denied troubleshooting.  
- Day 44: CloudWatch alarm not sending email (fix SNS).  
- Day 45: Document troubleshooting steps.  
- Day 46–50: Build support ticket examples (simulate customer issues).  
- Day 51–55: Practice solving tickets daily.  
- Day 56–60: Write GitHub “Support Ticket Solutions” section.

### Days 61–80: Portfolio Building
- Day 61–65: Build full project (EC2 + S3 + IAM + VPC).  
- Day 66–70: Add CloudWatch monitoring + alarms.  
- Day 71–75: Document project with screenshots + README.  
- Day 76–80: Polish GitHub repo, add clear structure.

### Days 81–90: Interview Prep
- Day 81–85: Review AWS services (EC2, S3, IAM, VPC, CloudWatch, CloudTrail).  
- Day 86–88: Practice common interview questions.  
- Day 89: Final GitHub + LinkedIn polish.  
- Day 90: Ready for Junior Cloud Support role.
